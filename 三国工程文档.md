# 《无双魏蜀吴》Flutter 工程框架设计文档

## 一、技术选型说明

### 1.1 为什么选 Flutter 做三国卡牌游戏

| 优势 | 说明 |
|------|------|
| 跨平台 | 一套代码支持 iOS / Android，节省成本 |
| UI 表现力 | 适合卡牌游戏大量 UI 界面（武将卡、阵容、商店） |
| 国风渲染 | 适合表现竹简、卷轴、印章等中国风元素 |
| 性能足够 | 回合制游戏 60FPS 足够 |
| 热更新友好 | 武将数值、活动配置可动态下发 |

### 1.2 核心技术栈

```
- Flutter 3.19+
- Dart 3.3+
- 状态管理：Riverpod 2.5+
- 网络：Dio + WebSocket（国战实时）
- 本地存储：Hive + SharedPreferences + SecureStorage
- 战斗引擎：Flame 1.16+（战斗场景） + Rive（武将动效）
- 路由：GoRouter
- 序列化：freezed + json_serializable
- 国际化：flutter_intl（预留繁中、英文、日文）
- 资源管理：flutter_gen
- 依赖注入：GetIt + Injectable
```

### 1.3 架构模式
**Clean Architecture + MVVM** 分层架构，结合 **DDD（领域驱动设计）** 思想。

## 五、数据流向设计

### 5.1 单向数据流

```
View (Page)
    ↓ 用户操作
ViewModel (Notifier)
    ↓ 调用
UseCase (业务逻辑)
    ↓ 调用
Repository Interface (领域层)
    ↓ 实现
Repository Impl (数据层)
    ↓ 调用
DataSource (远程/本地)
    ↓ 返回数据
Model → Entity (Mapper 转换)
    ↑ 数据回流
ViewModel 更新 State
    ↑
View 自动 rebuild
```

### 5.2 战斗模块特殊数据流

```
BattlePage (Flutter UI)
    ↓
BattleViewModel (Riverpod)
    ↓ 双向通信（命令/事件）
BattleController (Flame)
    ↓
BattleGame (Flame Game)
    ↓
各种 System (TurnSystem / SkillSystem / TacticSystem / DamageSystem / BondSystem)
    ↓
Components (武将、特效、UI)
```

### 5.3 国战实时数据流

```
KingdomWarPage
    ↓
KingdomWarViewModel
    ↓
WebSocketManager (KingdomWarChannel)
    ↓
Server (实时推送城池状态、战报)
    ↑
WebSocketManager
    ↑
ViewModel 更新 State
    ↑
KingdomWarPage 实时刷新
```
