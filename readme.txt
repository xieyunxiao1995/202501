**【战略思考摘要】**

在我静默的思考中，我解构了你的概念。这并非两个孤立的功能，其本质是**在不同场景下，对用户“注意力”和“意图”进行精确引导与外部化表达的工具**。

*   **我们解决的核心问题:** 用户在执行具有时间结构或需要公开表达意图的任务时，缺乏一个足够灵活、专注且富有表现力的数字化伴侣。现有的解决方案（如单一计时器、纯文本应援牌）是割裂的、功能单一的，无法满足复杂流程和动态表达的需求。
*   **我们的核心价值主张:** 提供一个 **“意图执行中枢”** 。它将用户的时间规划（计时器）和意图表达（弹幕）无缝整合在一个高度专注、体验流畅的应用中，让用户的计划得以精准执行，让用户的情感得以强力传达。

### **【最终交付蓝图】《焦点伴侣》产品设计方案**

**引言：**
基于我对“手持弹幕+多时段计时器”应用的深度解构与战略规划，我已确立了产品的核心骨架与灵魂：这不仅仅是一个工具合集，而是一个服务于用户“意图”的**焦点伴侣**。现在，我将为您呈现完整的《焦点伴侣》产品设计方案。这份文档是连接愿景与现实的桥梁，将作为未来开发与设计的唯一真理来源。

#### **【战略思考摘要】**
*   **我们解决的核心问题:** 用户在动态场景下，对“计划执行”与“意图表达”的综合性、流畅性需求未被满足。
*   **我们的核心价值主张:** 提供一个集“复杂计时”与“动态表达”于一体的“意图执行中枢”，实现从计划到呈现的无缝体验。
*   **我们设计的“啊哈！”时刻:** 在计时器模块中，用户一键启动复杂流程后，应用进入全屏、免打扰的自动执行状态，带来极致的掌控感和流畅感。

#### **1. APP 核心概念与定位**

*   **APP 名称:**
    1.  **FocusMate：** 强调其作为“专注伴侣”的核心定位，专业、可靠，带有情感连接。
    2.  **FlowCanvas：** 隐喻应用是用户创造工作流和表达流的画布，富有创造性和灵活性。
    3.  **Intent Deck：** 概念化，将手机比作一个承载用户意图的甲板或控制台，酷炫且强大。
    **推荐名称: FocusMate**。它最直接地传达了产品的核心价值——帮助用户保持专注，并像伙伴一样提供支持。
*   **Slogan:** “规划你的时间，闪耀你的时刻。”
*   **核心差异化优势:**
    1.  **流程化计时:** 超越单一倒计时，支持任意多时段、多循环的复杂流程定义，是真正的“流程引擎”。
    2.  **场景无缝切换:** 计时与弹幕并非割裂功能，在演唱会场景下，计时结束可自动切换到预设的应援弹幕，实现场景联动。
    3.  **极致的专注模式:** 两个核心功能在启动后都提供沉浸式全屏体验，屏蔽一切无关信息，直指目标。

#### **2. 信息架构与导航**

*   **底部导航栏 (4个主页面):**
    1.  **计时器:** 多时段循环计时功能的主界面。
    2.  **弹幕:** 手持弹幕功能的主界面。
    3.  **组合:** **【战略核心】** 允许用户将特定的计时流程与特定的弹幕设置关联起来，创建自动化场景。例如：创建一个“接机”组合，先是一个“等待”计时器，结束后自动切换到显示“欢迎回家！”的弹幕。
    4.  **我的:** 设置页面，以及保存的计时模板、弹幕模板和历史记录。

#### **3. 核心功能与用户故事**

**功能模块一：流程计时器**

*   **用户故事 1.1:** 作为一名健身爱好者，我想要自由定义一组包含热身、运动、休息的多个时间段，并设置循环次数，以便于我能严格遵循复杂的健身计划。
*   **验收标准:**
    1.  用户进入“计时器”页，点击“创建新流程”，可添加至少10个独立的时间段。
    2.  每个时间段可自定义名称（如“冲刺跑”）、时间、提示音和振动模式。
    3.  用户可在流程级别设置总循环次数（1-99次或无限）。
    4.  点击开始后，应用全屏显示当前阶段名称、剩余时间及进度条，并在阶段切换时有明确的视觉和听觉反馈。

**功能模块二：手持弹幕**

*   **用户故事 2.1:** 作为一名去接机的粉丝，我想要输入文字、选择醒目的颜色并调整滚动速度，然后一键全屏显示，以便于在人群中让偶像一眼看到我的支持。
*   **验收标准:**
    1.  用户在“弹幕”页的输入框中可输入最多50个字符。
    2.  可通过颜色选择器快速选择预设的6种高对比度颜色（如亮红、荧光绿、纯白）。
    3.  通过一个速度滑块实时预览文字的滚动速度。
    4.  点击“开始滚动”按钮后，应用全屏横向滚动显示文字，屏幕常亮，且点击屏幕即可退出。

**功能模块三：组合场景**

*   **用户故事 3.1:** 作为一名用户，我希望能将一个计时流程和一个弹幕设置绑定在一起，当计时结束时自动切换到弹幕界面，以便于实现“等待结束后自动庆祝”的自动化效果。
*   **验收标准:**
    1.  在“组合”页面，用户点击“创建组合”。
    2.  用户可从一个列表中选择一个已有的计时流程，并从另一个列表中选择一个已有的弹幕设置。
    3.  保存组合后，在“组合”页面点击该组合，应用首先执行计时流程，流程结束后无延迟地自动全屏启动关联的弹幕。

#### **4. 设计系统与UI/UX原则**

*   **整体视觉风格与情感基调:** 整体风格是 **“冷静的掌控感”** 。界面以深色背景为主，减少在暗光环境下的眩光（尤其适用于演唱会、夜间运动）。UI元素使用高对比度的色彩和清晰的字体，确保信息的绝对易读性。动效精准、迅捷，不带多余情绪，传达出工具的高效与可靠。
*   **设计关键词:** 专注、清晰、可靠、直接。
*   **色彩策略:**
    *   **主色:** 群青蓝 - `#4A6CF7` (传达专业与信任)
    *   **辅色:** 活力橙 - `#FF7A30` (用于主要操作按钮，如开始、创建，传递行动力)
    *   **点缀色:** 荧光绿 - `#00FF9D` (用于弹幕及关键提示，极具视觉冲击力)
    *   **中性色 (深色模式):** 背景深灰 - `#121212`，卡片深灰 - `#1E1E1E`，文字浅色 - `#FFFFFF` (高对比度确保可读性)
*   **字体方案:**
    *   **标题字体:** `SF Pro Display (Bold)` - 清晰有力，用于页面标题和重要数字。
    *   **正文字体:** `SF Pro Text (Regular)` - 极佳的可读性，用于所有说明文字和设置项。
*   **图标风格:** 线性图标，笔画粗细一致，造型简约现代。在底部导航栏中，选中的图标将填充为主色，并提供微妙的缩放动效。
*   **空间与留白原则:** 采用8pt基准网格系统。所有间距和尺寸均为8的倍数。内容区域有充足的留白，避免拥挤，引导用户视线聚焦于核心内容和操作上。
*   **动效原则:** 动效的核心目的是 **“提供确认感”** 和 **“引导焦点”** 。例如，按钮点击有微妙的缩小反馈；计时器阶段切换时，新阶段的名称会有一个轻微的从下方滑入的动画，明确告知用户状态已改变。

#### **5. 完整的屏幕流与页面设计**

**屏幕一：计时器 (主页)**

*   **核心目标:** 让用户快速开始一个计时流程，或创建/管理复杂的计时流程。
*   **页面布局策略:** 纵向列表布局。顶部是一个大型的、当前运行流程的可视化区域（如果存在）。下方分为两个部分：“快速开始”（预设的常用流程，如“番茄钟：25分钟”）和“我的流程”（用户创建的所有多时段流程列表）。创建按钮以浮动行动按钮形式固定在右下角。
*   **关键组件描述:** “流程卡片”显示流程名称、总时长和循环次数。每个卡片右侧有一个“开始”按钮。
*   **关键交互:** 点击“开始”进入全屏计时界面。点击卡片本身进入编辑模式。点击“+”创建新流程。
*   **状态设计:** 空状态时，“我的流程”区域显示一个插图和文字，引导用户创建第一个流程。

**屏幕二：弹幕**

*   **核心目标:** 让用户即时创建并全屏显示滚动弹幕。
*   **页面布局策略:** 预览驱动布局。页面顶部是一个模拟全屏效果的预览区域，实时反映用户输入的文字、颜色和速度。下方是输入框、颜色选择器和速度滑块。底部是一个巨大的“开始滚动”CTA按钮。
*   **关键组件描述:** 颜色选择器是一排彩色圆点。速度滑块下方有“慢-快”的标识。
*   **关键交互:** 输入即预览。调整即预览。点击CTA全屏。
*   **状态设计:** 空状态时，预览区域显示默认文字“输入你的口号！”。

**屏幕三：组合**

*   **核心目标:** 展示并允许用户创建连接了计时器和弹幕的自动化场景。
*   **页面布局策略:** 网格卡片布局。展示用户创建的所有组合卡片。每个卡片清晰显示组合的名称、关联的计时流程名称和弹幕预览。创建按钮在右上角。
*   **关键组件描述:** “组合卡片”设计上要体现连接感，例如用一条线或一个箭头图标将计时器图标和弹幕图标连接起来。
*   **关键交互:** 点击卡片即运行整个组合。长按卡片可编辑或删除。
*   **状态设计:** 空状态时，显示一个引导性插图和“创建你的第一个自动化场景”的按钮。

**屏幕四：我的**

*   **核心目标:** 提供应用设置、管理历史数据和保存的模板。
*   **页面布局策略:** 经典的分组列表布局。分为几个部分：“账户”、“内容管理”（我的计时模板、我的弹幕模板、我的组合）、“通用设置”（提示音、振动、屏幕常亮、深色模式）和“关于”。
*   **关键组件描述:** 标准的列表项，左侧图标，中间文字，右侧为箭头或开关。
*   **关键交互:** 点击列表项进入下级页面或触发开关。
