# 堆叠详情弹窗优化

## 问题
之前点击堆叠日记（如"平静时刻"）后，弹窗中的卡片高度固定为 220px，导致内容显示不全，用户无法看到完整的日记内容。

## 优化方案

### 1. 新的卡片布局
- 移除固定高度限制，改用自适应高度
- 使用更紧凑的卡片设计，专为列表展示优化
- 保留所有重要信息：日期、心情、图片、内容预览

### 2. 视觉改进
- **情绪色条**：顶部 4px 的渐变色条，快速识别心情
- **心情标签**：右上角显示心情状态（开心/平静/焦虑/忧郁）
- **图片预览**：120px 高度的图片，适合列表浏览
- **内容预览**：最多显示 3 行，超出部分用省略号
- **位置信息**：如果有位置，显示在底部左侧
- **操作提示**：底部右侧显示"点击查看详情"引导

### 3. 交互优化
- 整个卡片可点击，点击后关闭弹窗并跳转到详情页
- 保持流畅的动画效果
- 使用心情颜色作为视觉主题

### 4. 布局特点
```
┌─────────────────────────────┐
│ ████████████████████████    │ ← 情绪色条
│                             │
│  5月15日          [平静]    │ ← 日期和心情标签
│                             │
│  ┌───────────────────────┐  │
│  │                       │  │
│  │      图片预览         │  │ ← 120px 高度
│  │                       │  │
│  └───────────────────────┘  │
│                             │
│  今天开始整理我的书桌...    │ ← 内容预览（3行）
│                             │
│  📍 书房    点击查看详情 →  │ ← 底部信息
│                             │
└─────────────────────────────┘
```

## 效果
- ✅ 所有内容都能完整显示
- ✅ 卡片高度自适应，不会被截断
- ✅ 视觉层次清晰，信息一目了然
- ✅ 保持了原有的美观设计风格
- ✅ 交互流畅，用户体验更好

## 技术实现
- 创建了 `_buildStackCardItem()` 方法专门处理堆叠列表中的卡片
- 使用 `Column` 布局，让内容自然撑开高度
- 保留了动画效果和渐变色设计
- 统一了心情颜色的使用
